    # Vue Router

    # 单页应用

    # 概念

    SPA（Single Page Application）单页面应用程序，简称单页应用。指网站“所有”功能都在单个页面中进行呈现。如：管理系统、移动端、小程序

    # 优点

    • 前后端分离开发，提高了开发效率。
    • 业务场景切换时，局部更新结构。
    • 用户体验好，更加接近本地应用。

    # 缺点

    • 不利于 SEO。
    • 初次首屏加载速度较慢。
    • 页面复杂度比较高。

    # 前端路由

    # 概念

    前端路由，指的是 URL 与内容之间的映射关系

    # Hash 方式

    通过 window.onhashchange 事件监听 hash 变化，并进行网页内容更新。

    //封装的hash方式，以备复用
var router = {
    routes:{},
    route:function(path,callback){
        this.routes[path] = callback;
    },
    init:function(){
        var that = this;
        window.onhashchange = function(){
            var hash = window.location.href.replace('#','');
            that.routes[hash] && that.routes[hash]();
        }
    }  
}

//定义路由
route('/',()=>{
    container.innerHTML = '首页';
})

    # 特点总结

    • Hash 方式兼容性好。
    • 地址中具有#，不太美观。
    • 前进后退功能较为繁琐。

    # History 方式

    History 方式采用 HTML5 提供的新功能实现前端路由。

    通过 history.pushState() 变更 URL 并执行对应操作。

    通过 popState 事件监听前进后退按钮操作，并检测state

    //封装history方式，以便复用
var router = {
    routes:{},
    route:function(path,callback){
        this.routes[path] = callback;
    }
    go:function(path){
        //第一个参数为要保存的参数，路径，以便前进后退
        //第二个参数是要跳转的路径
        history.pushState({path : path},null,path);
        this.routes[path] && this.routes[path]();
    },
        //初始化前进后退按钮
    init:function(){
        var that = this;
        window.addEventListener('popstate',function(e){
            var path = e.state ? e.state.path:'/';
            that.routes[path] && that.routes[path]();
        })
    }
}
//初始化
router.init()

//定义路由
route('/',function(){
    container.innerHTML = '首页'
});

//调用
var links = document.querySlectAll('a');
var container = document.querySlect('#container');
links.forEach(ele=>{
    ele.onclick = function(e){
        var path = e.target.getAttribute('href');//用getAttribute才能只获取/user/部分
        router.go(path);
        return false;
    }
})

    # Vue Router

    是Vue.js 官方的路由管理器,让构建单页面应用变得易如反掌。

    # 基本使用

    • Vue Router 提供了用于进行路由设置的组件<router-link>和<router-view>
    <router-link to="/">首页</router-link>
<router-link to="/category">分类</router-link>
<router-link to="/user">用户</router-link>
<router-view></router-view>
    • 定义路由中需要的组件，并进行路由规则设置。
    var Index = {
    template: `<div>这是首页</div>`
    };
var Category = {
    template: `<div>这是分类</div>`
};
var User = {
    template: `<div>这是用户</div>`
};
var routes = [
    { path: '/', component: Index },
    { path: '/category', component: Category },
    { path: '/user', component: User }
];
    • 创建 Vue Router实例,通过 routes属性配置路由。
     const router = new VueRouter({
        routes
    });
    • 创建 Vue 实例，通过router 属性注入路由。
    const vm = new Vue({
        el: '#app',
        router
    })

    # 命名视图

    • 如果导航后，希望同时在同级展示多个视图(组件），这时就需要进行命名视图。
    <router-link to="/">首页</router-link>
<router-link to="/user">用户</router-link>

<router-view name="sidebar"></router-view>
<router-view></router-view>
    • 路由中通过components属性进行设置不同视图的对应组件。
    var routes = [
    { path: '/', components: { default: Index, sidebar: SideBar } },
    { path: '/user', components: { default: User, sidebar: SideBar } },
];

    # 动态路由

    当我们需要将某一类URL都映射到同一个组件，就需要使用动态路由。

    • 定义路由规则时，将路径中的某个部分使用:进行标记，即可设置为动态路由。
    var User = {
        template: `<div>这是用户{{$route.params.id}}的信息</div>`
    };
var routes = [{path:'/user/:id',component:User}];
    • 设置为动态路由后，动态部分为任意内容均跳转到同一组件。
    • : 部分对应的信息称为路径参数，存储在 vm.$route.params 中

    # 侦听路由参数

    当使用路由参数时，例如从 /user/foo 导航到 /user/bar，原来的组件实例会被复用。这也意味着组件的生命周期钩子不会再被调用。

    复用组件时，想对路由参数的变化作出响应的话，你可以简单地 watch (监测变化) $route 对象：

    var User = {
    template: `<div>这是用户{{$route.params.id}}的信息</div>`,
    // 由于不会重新创建组件，参数可通过侦听的方式进行获取
    watch:{
        $route(route){
            console.log(route.params.id);
        }
    }
};

    # 路由传参处理

    • 这里通过路由的props 设置数据,并通过组件props接收。
    var Category = {
        props:['id'],
        template:`<div>这是分类{{ id }}的内容</div>`
    }
var routes = [
    {path:'/category/:id',component:Category,props:true}
];
    • 包含多个命名视图时，需要将路由的 props 设置为对象。
    • 如果希望设置静态数据，可将 props 中的某个组件对应的选项设置为对象，内部属性会绑定给组件的 props。
    var Category = {
    props: ['id'],
    template: `<div>这是分类{{ id }}的内容</div>`
}
var SideBar = {
    template: `<div>这是侧边栏内容</div>`
}
var SideBar1 = {
    props: ['a', 'b'],
    template: `<div>这是侧边栏内容{{a}},{{b}}</div>`
}
var routes = [
    { path: '/user/:id', component: User },
    { path: '/category/:id', 
     components: { default: Category, sidebar: SideBar,sidebar1:SideBar1 },        props: { default: true ,sidebar1:{a:'我是aaa',b:'我是BBB'}} }
    //default通过props传值，sidebar1通过props传递静态参数
];

    # 嵌套路由

    实际场景中，路由通常由多层嵌套的组件组合而成，这时需要使用嵌套路由配置。

    • 使用 children 来进行嵌套路由中的子路由设置
    • 在Vue Router 中/代表根目录
    var routes = [
        {
            path:'/user',
            component:User,
            children:[
                {
                    path:'sex',
                    component:Sex
                },{
                    path:'info',
                    component:Info,
                    children:[
                        {
                            path:'school',
                            component:InfoSchool
                        }
                    ]
                }
            ]
           
        }
    ];

    # 编程式导航

    编程式导航，指的是通过方法设置导航

    • router.push() 用来导航到一个新 URL
    vm.$router.push('/user');
vm.$router.push({path:'/user'});
    • <router-link> 的 to 属性使用绑定方式时也可以写成属性对象结构
    • 注意：使用path时，params会失效。可改变方式
      • 直接写全path
      • name+params
    <router-link :to = "{path : "/user/10"}">用户10</router-link>

    # 命名路由

    • 设置路由时添加name属性。
    var School = {
template:`<div>School 组件 {{ $router.params.id }}</div>`
};

var routes = [
    {
    	path:'/user/:id/info/school',
		name:'school',
		component:School
    }
];
    • 在 push() 中通过 name 导航到对应路由，参数通过 params 设置。
    vm.$router.push('school',params:{id:10});
    • 也可以在<router-link>中使用
    <router-link :to = "{name:'school',params:{id : 10}}"></router-link>

    # 其他功能

    # 重定向

    在路径不合法时，可设置重定向.

    “重定向”的意思是，当用户访问 /a时，URL 将会被替换成 /b，然后匹配路由为 /b

    //示例代码
var routes = [
    {
        path:'/',
        component:Index
    }
    ,
    {
        path:'/category/:id',
        component:Category
    },
    {
        path:'/category',
        redirect:'/'
    }
]

    # 别名

    /a 的别名是 /b，意味着，当用户访问 /b 时，URL 会保持为 /b，但是路由匹配则为 /a，就像用户访问 /a 一样。关键字：alias

    //视图中的用法
//原始name用法
<router-link :to="{name:'school',params:{id:10,dete:0621}}"></router-link>
//别名用法
<router-link to = "/10/0621"></router-link>

    //别名定义
var router= {
    path:`/user/:id/info/school/intro/:date`,
    name:'school',
    component:School,
    alias:'/:id/:date'
}

    # History 模式

    需要通过 Vue Router 实例的 mode 选项来设置，这样 URL 会更加美观，但同样需要后端支持避免问题。

    var router = new VueRouter({
    mode:'history',
    routes
});
